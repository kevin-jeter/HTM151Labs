User Guide

Welcome to the HTM151: Nuvolo for Managers Labs! This guide is designed to help you navigate through the various labs included in this course. Each lab focuses on a specific aspect of using Nuvolo, providing hands-on experience and practical knowledge.

Getting Started

  1. Access Nuvolo Development Environment: Use the DEV environment for all labs. Login via https://vaoittest.servicenowservices.com with your VA.gov account.
  2. Navigate This Guide: Use the dropdown menu above to select a specific lab. All sections include step-by-step instructions, screenshots, and interactive elements like input fields for answers.
  3. Theme Toggle: Switch between light and dark mode using the toggle in the top right for better readability.

Tips for Success

  • Explore Freely: Don't hesitate to click around Nuvolo—use the "History" tab to backtrack if needed. Avoid using your browser's back button, as it may not work well with Nuvolo.
  • Document Your Work: Many labs include input fields for recording answers (e.g., work order numbers). Use these to track your progress and share with instructors.
  • Troubleshooting: If you encounter issues, you're not alone! Ask your instructor(s) for help and you'll be helping out your peers too!
  • Practice Hypotheticals: Labs encourage creating hypothetical work orders and assets to build confidence without real-world consequences.

If you have questions, reach out to your instructor or refer to the course materials. Happy learning and mastering Nuvolo!

Happy learning!

Lab 1: Supervisor Foundations

In this lab, you will learn how to enable User Attributes, set Technician Schedules and check Internal Rates.

Follow the steps in each part. You’ll practice the exact click-paths used in class and answer short knowledge-checks at the end of each part. Type your answers in the spaces provided.

Part 1: User Attributes

  1. Navigate to User Attributes. Select All in the Filter Navigator. Type user in the search bar and select Nuvolo User Attributes (under Clinical Asset Management Admin).
  2. Click the Show/Hide Filter (funnel) button. Remove all conditional filters by selecting the red X buttons. Click Run.
  3. Click the Show Column Search Row button. Type your name in the Name column search box. Press Enter.
  4. Select your Name (by selecting the blue hyperlinked name) and review the following data fields for accuracy:
    • Facility: The facility you report to on an organizational chart
    • Duty Station: The facility you are performing work actions at currently
    • HTM User. Make sure this box is checked to ensure you have access to Nuvolo
    • HTM Manager. Make sure this box is checked to ensure you have manager privileges in Nuvolo
    • HTM Primary Role. Your user role, as applicable
  5. Once confirmed, select "Update" (only necessary if you make changes)
  6. Repeat this steps for at least 2 additional technicians at your site. Which technicians did you update?
  7. Which two data fields under the New Section tab do you have the option to put a check mark?

    8. A: HTM User and HTM Manager

  8. Who is responsible for enabling HTM user profiles?

    9. A: Supervisors/HTM Managers

  9. Which checkbox grants access to Nuvolo as an HTM user?

    10. A: HTM User (and HTM Manager if applicable).

Part 2: Technician Schedules

  1. In the "All" filter navigator, search for "Technician Schedules" and select it.
  2. We're going to create a New Technician Schedule for the technicians you updated User Attributes for. Select the "New" button.
  3. Fill out the following fields on the form using one of your technicians.
    • Name: Technician Name
    • Technician: Technician Name (Use the magnifying glass to search)
    • Schedule Type: Shift
    • Description: HTM151 Technician Schedule Example
  4. Repeat these steps for the other technician you updated User Attributes for.
  5. Let's find the schedule! In the "Name" search bar, enter "*" and then the last name of either of the technicians you created.
  6. Select your Technician Schedule from the list and click on "Load Related Lists" at the bottom of the page.
  7. Let's create a new Schedule Entry. Click on "New" in the Schedule Entries related list.
  8. Fill out the following fields on the Schedule Entry form:
    • Name: Technician Name Monday-Friday
    • Type: --None--
    • Show as: Busy
    • When: Today's Date and Their Work Hours
    • Repeats: Every Weekday
    • Repeat Until: Leave Blank
    • Type: Leave Blank
  9. Submit the Schedule Entry. You'll see the schedule entry appear in the related list.
  10. Repeat these steps for the other technician.
  11. What happens to workorders if a technician has no active schedule?

    12. A: Work orders route to the assignment group

  12. Where do you add the daily time range for a schedule?

    13. A: In the Schedule Entry (When section) or use the All‑day checkbox.

Part 3: Rates

Let’s make sure you know how to accurately view and update your technician’s pay rates within Nuvolo. This ensures proper record keeping to see how much labor cost is being put into specific devices. It will help make decisions on device maintenance and upkeep versus device replacement.

  1. In the "All" filter navigator, search for "Internal Employee Rates" and select it.
  2. Let's create a new Employee Rate. Select the New button.
  3. We're going to create a new employee rate. Fill out the following fields on the Employee Rate form:
    • Employee: One of the technicians names
    • Position Entry Date: Today's Date
    • Facility: Your Facility
  4. Submit the Employee Rate.
  5. Repeat for the other technician
  6. Search and find the first employee rate you just created using the Employee search (Hint: type *Jeter)
  7. Open the entry by selecting the blue hyperlink under "Yearly."
  8. We need to calculate Kevin's Labor rate. To do so, follow these steps:
    • Annual Salary: $40,000
    • Divide this by 2080: $40,000 / 2080 = $19.23 (round to the nearest cent)
    • Enter $19.23 into the "Labor Rate" field
  9. Finally, let's fill out the rest of Kevin' Internal Rate Information. Fill out or confirm the following fields:
    • Yearly: Enter value from MSS
    • Pay Plan: Enter GS
    • Classification Series: Enter the job series. (0858 is for BME)
    • Grade: Enter Pay Grade
    • Step: Enter current pay grade step
    • Role: Enter HTM Role
  10. Select "Update" and you've completed updating the employee rate.
  11. Repeat these steps for the other technician.
  12. Where do rate values originate?

    13. A: Manager Self Service (MSS)

  13. Who can obtain the required values if you cannot access MSS?

    14. A: Supervisory BESSs/BMEs or your VISN chief

Lab 2: Clinical Device Verification and Work Order Assignment

Part 1: Navigation and Primary Data Fields

  1. In the "All" filter navigator, search for "Unverified Devices" and select it. You may need to add "Verified" as a column to see the unverified devices.
  2. Click on the first Unverified Device to view its details. Hint: You must select the Asset Tag to open the device!
  3. Review the primary data fields on the device details page. Based on the information, answer the following questions:
    1. What is the device's serial number?
    2. What is the Operational Status of this device?
    3. Is there a Backup Available?
    4. Is this device a Loaner?
    5. What is the Warranty Expiration date?
    6. What is the PM Support Group?
    7. What was the last activity on this device?
    8. Is this device Cerner compatible?
  4. Before verifying this device, let's find out how to automatically assign groups for PM and CM Support on this device. Select the "Resources" tab.
  5. Let's update the PM Support Group. Select the "Eyeglass" icon next to the PM Support Group field.
  6. In the pop-up window, select one of the groups to give them automatic assignment to this device.
  7. Finally, let's verify this device. Select the "Verified" button and select Save.
  8. What time does the Daily Device Feed run every day?

    9. A: 2 AM EST

  9. Where is device data pulled from to the daily device feed?

    10. A: Device data is pulled from AEMS/MERS (VistA), Maximo, NMDD

  10. Where can you see the Daily Device Feed update history within a device record?

    11. A: Under the Activities tab at the bottom of the Clinical Devices record

Part 2: Work Order Assignment

We just used Automatic Assignment in the last lab, but how do we manage our groups and create new ones if we need to?

  1. In the "All" filter navigator, search for "Groups" and select it (Hint: It's the one under "Self Service")
  2. In the list, search for "HTM" followed by the HTM Program TLA (in this example, we use ABQ. Yours will be different!) and select Enter
  3. Select one of the groups.
  4. At the bottom of the page, you can select "Load Related Lists" to view additional details like Roles, Group Members, Groups, Skills, etc.
  5. Review the data fields and answer the following questions:
    1. What is the purpose/description of this group?
    2. How many Group Members are in this group?
    3. How many different Roles are assigned to this group?
    4. Is the group Active?
    5. What is the Group email?
  6. What is the role of the Group Manager in a Support Group?

    7. A: Typically the HTM Manager or department chief responsible for overseeing the group’s settings and performance

  7. What function does the Default Assignee serve in a Support Group?

    8. A: It determines the automatically assigned technician if no specific Supported By person is listed on the device

  8. Why is accurate group membership critical for automatic assignment?

    9. A: Nuvolo uses Support Group membership to route CM, PM, Cyber, and Other work orders to the correct personnel

  9. Why should you check your technician’s schedule before relying on automatic assignment?

    10. A: Because automatic assignment only works when a technician has an active schedule; otherwise, work orders default to the group instead.

Lab 3: Preventive Maintenance Definitions

A key aspect of PMs is PM definitions. This is how PMs will be auto-generated within Nuvolo. For this lab, we'll focus on creating a PM definition for "CAPNOMETERS"

Part 1: Maintenance Definitions and Automatic Creation

  1. In the "All" filter navigator, search for "Maintenance Definitions" and select it.
  2. Select "New" to create a new PM Definition.
  3. Fill out the following fields:
    • Name: VISN 20 - WWW - CAPNOMETERS - CAPNOCHECK II
      • Note: Replace VISN with your correct VISN and "WWW" with your three-letter HTM Program TLA
    • VISN: Choose your VISN
    • Facility: Remove this Filter for our Labs. You'll want this when creating them normally though!
    • Model Name: CAPNOCHECK II
    • Status: In Use, Loaned Out, Out of Service
  4. Scroll down for the next section to fill out. Follow the screenshot below for all fields:
  5. Select "Save" or "Submit" to complete the PM Definition creation.
  6. Let's find your PM Definition in the list. One of the easiest ways to do so is by editing your columns and searching by "Created by"
    • Select the gear icon in the top right corner of the list view
    • Search for "Created by" and add it to your selected columns
    • In the column Search, type "*" and your Last Name to find your PM Definition
  7. Select your newly created definition by clicking on the name
  8. Confirm the PM Definition is activated. If it isn't, click "Activate" in the top right corner.
  9. Why won’t Clinical Scheduled Maintenances appear before clicking Activate?

    10. A: Can only be generated only after the definition is activated.

  10. Where do you see the number of matching assets for your conditions?

    11. A: In the Asset Conditions in the link: No. of records matching the conditions

  11. What are the two related lists you should verify after activation?

    12. A: Maintenance Instance and Upcoming Scheduled Maintenance in Load Related List
